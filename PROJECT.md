# Мир эмоций — Проектная документация

## Выпускная квалификационная работа

**Тема:** «Разработка и анализ эффективности игрового мобильного приложения для развития навыка распознавания базовых эмоций у школьников с расстройством аутистического спектра»

Приложение «Мир эмоций» — мобильная игровая обучающая программа, разработанная в рамках выпускной квалификационной работы. Целевая аудитория — школьники с расстройством аутистического спектра (РАС), испытывающие затруднения в распознавании базовых эмоций на лицах других людей. Педагог или родитель сопровождает ребёнка в работе с приложением и получает аналитику в реальном времени.

---

## Предметная область

### Теоретическая база

Приложение опирается на классификацию базовых эмоций Пола Экмана (шесть универсальных эмоций: радость, грусть, гнев, страх, удивление, отвращение) и методику социальных историй Кэрол Грей (Social Stories™). Все три обучающих модуля выстроены по принципу нарастающей когнитивной нагрузки: от прямого узнавания фотографий до понимания эмоционального контекста ситуации.

### Обучаемые эмоции

| ID           | Название     | Emoji | Мимические признаки                                    |
| ------------ | -------------------- | ----- | ------------------------------------------------------------------------ |
| `joy`      | Радость       | 😄    | Приподнятые брови, широкая улыбка           |
| `sadness`  | Грусть         | 😢    | Уголки рта опущены, брови сдвинуты          |

## Дизайн исследования

Приложение реализует **квазиэкспериментальную схему** с одной группой и двумя измерениями:

```
Констатирующий этап (pre-test)
        ↓
Обучающий блок (3 модуля, адаптивная сложность)
        ↓
Контрольный этап (post-test)
        ↓
Сравнительный анализ (встроенная аналитика + CSV-экспорт)
```

**Метрики эффективности:**

- Точность распознавания (accuracy) по каждой эмоции и в целом
- Среднее время реакции (reaction time, мс)
- Динамика точности по сессиям в процессе обучения
- Сравнение pre/post по каждому участнику

## Персонаж-помощник

В приложении есть персонаж — **кот Апельсин** (🐱). Имя задано в онбординге явно: «Привет! Я — кот Апельсин!» (`onboarding_screen.dart:25`).

Апельсин появляется в трёх контекстах:

1. **Онбординг** — первый слайд знакомит ребёнка с персонажем и объясняет механику игры.
2. **Подсказки в модулях** — виджет `HelperCat` (`common_widgets.dart:134`) отображает 🐱 с речевым пузырём, в котором написана подсказка к текущему заданию (поле `helperHint` из `Module1Task`). Пузырь появляется до ответа и скрывается после (`AnimatedSize`).
3. **Экран награды** — текст «Апельсин очень рад за тебя 🐱» (`reward_screen.dart:44`).

Апельсин также используется как эмодзи в баннере предупреждения таймера (`_TimerWarningBanner`) и на handoff-экране конца сессии, но там без имени.

**Нарратива и сюжетной линии нет** — персонаж выполняет исключительно роль функционального помощника (подсказчик + эмоциональное поощрение).

## Обучающие модули

### Модуль 1 — «Знакомство» (`/module1`)

**Задача:** Выбрать правильную эмоцию из нескольких карточек с фотографиями реальных людей.

- 18 заданий в пуле: 6 эмоций × 3 варианта фото (`imageIndex` 0–2)
- Задания фильтруются по сложности: показываются только задания с `difficultyLevel <= ((maxDiff + 1) ~/ 2 + 1)`, что при начальном уровне 1 ограничивает пул лёгкими и средними заданиями
- Число вариантов ответа адаптивно: 2 / 4 / 6 (из `getChoiceCount`)
- Состав карточек: 1 целевая эмоция + (N−1) случайных дистракторов из оставшихся 5, перемешанных каждый раз заново
- Карточки отображаются в **сетке 2 колонки** (фиксированный `crossAxisCount: 2`)
- Фото рендерятся через `ColorFilter` (матрица смягчения насыщенности) для снижения интенсивности изображения
- До ответа: персонаж Апельсин показывает подсказку (`HelperCat` с `helperHint`)
- После ответа: `HelperCat` скрывается через `AnimatedSize`; появляются `FeedbackBanner` (зелёный/красный), `EmotionInfoCard` с описанием эмоции и кнопка «Дальше»
- Правильная карточка подсвечивается зелёной рамкой; ошибочно выбранная — красной

### Модуль 2 — «Конструктор» (`/module2`)

**Задача:** Собрать эмоцию по частям лица — выбрать правильное описание бровей и рта.

- Два независимых выбора: «брови» и «рот»
- Пул текстовых вариантов для бровей и рта (4 опции каждый)
- Задание считается выполненным при верном выборе обеих частей
- Визуальная иллюстрация — фото целевой эмоции
- Дополнительная интерактивность: выбор «взрослого» или «детского» описания (`_AdultView` / детское представление)

### Модуль 3 — «Эмоции в ситуации» (`/module3`)

**Задача:** Прочитать социальную историю и определить, что чувствует персонаж.

- Основан на методике Social Stories™ (Кэрол Грей)
- Текст ситуации описывает конкретный жизненный эпизод
- Выбор из нескольких эмоций-ответов
- Акцент на понимании эмоционального контекста, а не только мимики

## Адаптивная сложность

Каждый модуль имеет независимый уровень сложности (1–5):

```
Скользящее окно последних 5 ответов (максимум)
    — пересчёт не запускается, пока в окне < 3 ответов
    → accuracy > 85%  →  level + 1  (до максимума 5)
    → accuracy < 70%  →  level - 1  (до минимума 1)
    → иначе           →  level без изменений
```

**Маппинг уровня → число вариантов ответа (Модуль 1):**

| Уровень сложности | Число вариантов |
| --------------------------------- | ----------------------------- |
| 1–2                              | 2 варианта            |
| 3–4                              | 4 варианта            |
| 5                                 | 6 вариантов          |

При первом входе после прохождения пре-теста начальный уровень сложности **инициализируется из точности диагностики** (`seedDifficultyFromAccuracy`), чтобы обучение начиналось на соответствующем уровне, а не всегда с первого.

## Диагностическая система

### Две методики измерения (ROADMAP задача #12)

Батарея измеряет **два компонента** распознавания эмоций (`DiagnosticMeasure`):

1. **«Эмоциональные лица» (`faces`)** — перцептивный компонент: распознавание эмоции по фотографии лица. 18 заданий (3 на эмоцию). Основная метрика — на ней строится сравнение pre/post и сидирование стартовой сложности модулей.
2. **«Социальные истории» (`stories`)** — контекстный компонент: понимание эмоции по описанию ситуации (методика Social Stories™ Кэрол Грей). 6 заданий (1 на эмоцию), текст без фото, ответ теми же эмодзи-карточками эмоций.

Итого батарея — **24 задания**, предъявляются двумя блоками (сначала все лица, потом все истории). Сюжеты историй **новые**, не пересекаются с обучающими `module3Tasks` — иначе натренированный ребёнок узнавал бы сюжет, а не понимал эмоцию (leak обучения в измерение).

Метрики раздельны: `DiagnosticSession.accuracy` / `avgReactionMs` / `accuracyForEmotion` считают **только faces**; для историй — `storiesAccuracy` / `storiesAvgReactionMs`. Каждый `DiagnosticAnswer` несёт поле `measure` (старые сессии → `faces`). CSV: детальные экспорты — колонка `measure`; сводные — `pre_stories_accuracy`/`post_stories_accuracy`.

Обе методики имеют параллельные формы A/B и контрбалансируются одним правилом (см. ниже).

### Структура теста (методика «Эмоциональные лица»)

- **18 заданий: по 3 на каждую эмоцию Экмана** (ранее было 6 — расширено в задаче #2 ROADMAP для статистической чувствительности)
- Всегда 4 варианта ответа
- Дистракторы подобраны по принципу максимальной близости мимики (например, страх ↔ удивление, гнев ↔ отвращение)
- **Все 3 задания одной эмоции используют одинаковый набор дистракторов** — это сохраняет психометрическую стабильность: каждое задание измеряет одну и ту же пару путаниц, что важно для McNemar и других тестов парных категориальных данных
- Каждое задание владеет своим стимулом через поле `DiagnosticTask.imagePath` — 3 задания на одну эмоцию используют разные фотографии лиц
- Интерливинг: задания перетасовываются так, чтобы одинаковые эмоции не шли подряд (жадный алгоритм в `diagnostic_screen._buildShuffledTasks`)
- Стимульные фото лежат в `assets/images/diagnostic/form_a/<emotion>_<N>.jpg` и `form_b/<emotion>_<N>.jpg` (две параллельные формы — см. ниже) и не пересекаются с обучающими — это **критично методологически**: иначе измеряется запоминание конкретного лица, а не обобщённое распознавание эмоции

### Почему расширили с 6 до 18

Прежний формат (6 заданий, 1 на эмоцию) имел три проблемы для ВКР:

1. **Per-emotion accuracy была бинарной** (0 или 1) — невозможно применить McNemar на одном задании, тонкая динамика по эмоциям не фиксировалась.
2. **Общая точность кратна 1/6 (~17%)** — нельзя различить ребёнка с «5/6» и «6/6», хотя для гипотезы это важно.
3. **Любая случайная ошибка** (отвлёкся, ткнул мимо) давала −17 п.п. — большое искажение измерения.

После расширения: per-emotion accuracy кратна 1/3 (0.00 / 0.33 / 0.67 / 1.00), общая точность кратна 1/18 (~5.5%), вес случайного промаха — ~5.5%.

### Параллельные формы и контрбалансировка (ROADMAP задача #3)

Диагностическая батарея существует в двух параллельных формах: `diagnosticTasksFormA` и `diagnosticTasksFormB`. Структура форм идентична (те же 6 эмоций × 3 задания, те же дистракторы, тот же гендерный баланс), различаются только **стимульные фото** — Form A использует одних актёров KDEF, Form B — других. Всего во всём материале **36 уникальных актёров**, без пересечений ни внутри форм, ни между ними.

**Зачем две формы.** Прежний дизайн (одна батарея, одни и те же фото в pre и post) давал конкурирующую гипотезу: улучшение результатов после обучения можно было объяснить тем, что ребёнок просто **запомнил правильные ответы** на конкретные фото за два прохождения, а не научился распознавать эмоции в принципе. Это критический test-retest learning effect, который рецензент ВКР почти наверняка укажет.

**Зачем контрбалансировка.** Самое простое решение «pre = всегда Form A, post = всегда Form B» создаёт новый confound: если Form B окажется случайно чуть труднее (или легче) Form A, разница в результатах будет приписана обучению, хотя на самом деле это эффект формы. Контрбалансировка устраняет это: у половины детей pre=A/post=B, у другой половины — pre=B/post=A. Любая систематическая разница форм усредняется по выборке.

**Реализация.** Функция `getDiagnosticTasks(phase, participantId)` назначает форму по чётности `participantId.hashCode`:

- чётный hash → pre=A, post=B
- нечётный hash → pre=B, post=A

Назначение детерминированное — ребёнок при повторном входе в ту же фазу увидит ту же форму. Это важно при технических сбоях или при необходимости пройти диагностику в два захода. Источник: `lib/data/diagnostic_tasks.dart`.

Методика «Социальные истории» тоже имеет две параллельные формы (`socialStoryTasksFormA` / `socialStoryTasksFormB`, по 6 сюжетов) и контрбалансируется тем же правилом, что и лица — `getDiagnosticTasks` возвращает faces и stories одной и той же формы.

**Стимульный материал.** Все фото взяты из датасета KDEF (Karolinska Directed Emotional Faces; Lundqvist, Flykt & Öhman, 1998) — валидированный для психофизиологических исследований набор фронтальных портретов с 7 эмоциями × 5 ракурсами для 70 актёров. Используется только фронтальный ракурс (`S` в имени файла KDEF), 36 разных людей, баланс мужчин/женщин 18/18. Расположение: `assets/images/diagnostic/form_a/<emotion>_<N>.jpg` и `form_b/<emotion>_<N>.jpg`.

### Сбор данных

Каждый ответ сохраняется как `DiagnosticAnswer`:

- `emotionId` — целевая эмоция
- `selectedId` — выбранная участником эмоция
- `isCorrect` — верность ответа
- `reactionTimeMs` — время реакции в миллисекундах
- `timestamp` — момент ответа

### Экспорт CSV

В CSV выгружаются **только диагностические сессии** (`DiagnosticModel`). Обучающие метрики из `ProgressModel` (прогресс модулей, кумулятив по эмоциям, активность по дням, адаптивная сложность, время реакции в обучении) **в CSV не попадают** — они доступны только в UI на экране `/analytics`. Если потребуется выгружать обучающую часть для статистики — нужно добавлять отдельные экспортёры.

Доступны **4 формата** — это пересечение двух осей: гранулярность × охват.

|                                                                | Один профиль                             | Все профили на устройстве                              |
| -------------------------------------------------------------- | --------------------------------------------------- | ---------------------------------------------------------------------------- |
| **Детальный** (1 строка = 1 ответ)   | `exportCsv()` → `answers_<id>_<ts>.csv`        | `exportAllProfilesCsv(profiles)` static → `all_answers_<ts>.csv`        |
| **Сводный** (1 строка = 1 участник) | `exportSummaryCsv()` → `summary_<id>_<ts>.csv` | `exportAllProfilesSummaryCsv(profiles)` static → `all_summary_<ts>.csv` |

#### Колонки

**Детальный** (10 / 11 колонок):

```
participant_id, [participant_name,] group, phase, date, trial_number,
measure, emotion_id, selected_id, is_correct, reaction_time_ms
```

- `participant_name` — только в варианте «все профили».
- `group` — `experimental` / `control`.
- `phase` — `pre` / `post`; `measure` — `faces` / `stories`; `is_correct` — `0` / `1`.

Назначение: матрица смешения, хронология ответов, временной анализ латентности, Мак-Немар по парным ответам.

**Сводный** (40 / 41 колонка). Схема собирается из общих хелперов `DiagnosticModel._summaryHeaderTail` / `_summaryRowTail` — менять её нужно только там (обе сводные формы строятся из них). Порядок эмоций фиксирован: `joy, sadness, anger, fear, surprise, disgust`.

```
participant_id, [participant_name,] group,
pre_accuracy, post_accuracy,                       # точность ЛИЦА, доля 0..1
pre_avg_rt_ms, post_avg_rt_ms,                     # среднее RT по лицам, мс
pre_stories_accuracy, post_stories_accuracy,       # точность ИСТОРИИ, доля 0..1
pre_faces_correct, pre_faces_total,                # сырые счётчики лиц (балл 0..18)
post_faces_correct, post_faces_total,
pre_stories_correct, pre_stories_total,            # сырые счётчики историй (балл 0..6)
post_stories_correct, post_stories_total,
pre_<emotion>_acc  × 6,  post_<emotion>_acc  × 6,  # точность по эмоции (лица), доля
pre_<emotion>_rt_ms × 6, post_<emotion>_rt_ms × 6  # RT по эмоции (лица), мс
```

- `accuracy`/`*_acc` — доли `0.00`–`1.00` (`toStringAsFixed(2)`); базовые `pre/post_accuracy` = **только лица** (перцептивный компонент).
- RT — целые мс; счётчики — целые (восстановить per-emotion счётчик: `<emotion>_acc × 3` для лиц).
- Нет пары pre/post (пройдена одна фаза) — недостающие значения = `0`.

Назначение: готовый формат под критерий Вилкоксона (одна строка = пара до/после), плюс сырьё под Мак-Немар и любую балльную шкалу. Полный разбор переменных и методов — раздел [«Статистика проекта»](#статистика-проекта).

#### Охват

- **Один профиль** (instance-методы) — работает с уже загруженным в память `_sessions`. Используется педагогом для индивидуального отчёта по конкретному ребёнку.
- **Все профили** (static-методы) — читает данные всех профилей напрямую из `SharedPreferences` по ключам `profile_<id>_diagnostic_sessions`, не требует загружать профили по одному. Используется для агрегатной выгрузки выборки ВКР.

Все 4 формата открываются через нативный share sheet iOS (`share_plus`) — файл пишется во временную директорию app-sandbox и передаётся системе для сохранения в «Файлы», AirDrop, почту и т.д. На desktop пишет напрямую в `~/Documents` и показывает snackbar с путём.

UI-кнопки на handoff-экране (`lib/screens/handoff_screen.dart`, `_buildExportCard`):

- секция «текущий ребёнок» — два варианта по активному профилю;
- секция «все дети» — два варианта по всем профилям.

## Таймер сессии

Защищает ребёнка от переутомления:

- Настраиваемая продолжительность (по умолчанию 10 минут; сохраняется в `shared_preferences`)
- За 2 минуты до конца — анимированный баннер-предупреждение `_TimerWarningBanner`, автоматически исчезает через 6 секунд
- По истечении времени — приложение **полностью прерывает занятие**: автоматический переход на handoff-экран в режиме `HandoffMode.sessionExpired` (маршрут `/session_end`). Ребёнок видит мягкое сообщение «Время отдохнуть!» и просьбу передать устройство педагогу; педагог после ввода PIN видит сводку текущего прогресса по модулям и возвращается на главный экран. Прогресс к моменту прерывания уже сохранён автоматически (каждый ответ записывается в `ProgressModel` в момент `submitAnswer`, индекс задания — в `advanceTask`)
- Таймер стартует при входе в любой модуль (`startSessionTimer` в `ModuleTaskMixin`)

## Аналитика

Экран `/analytics` защищён PIN и содержит три вкладки.

### Источники данных

Аналитика читает две независимые модели через Provider:

- **`ProgressModel`** — обучающие метрики (заполняются в модулях 1–3 через `recordAnswer`). Только для отображения в UI, **в CSV не экспортируются**.
- **`DiagnosticModel`** — пары диагностических сессий pre/post. Используются для сравнения «до/после» в UI и **являются единственным источником CSV-выгрузок** (см. раздел «Экспорт CSV» выше).

### Полный перечень собираемых метрик

#### Из `ProgressModel` (хранятся на профиль с префиксом `profile_<id>_`)

| Поле                                  | Тип                                                                          | Где видно                                                                               |
| ----------------------------------------- | ------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `moduleProgress`                        | Map<String,double> 0–1 на модуль                                       | вкладка «Обучение» → «Прогресс по модулям»                 |
| `correctByEmotion` / `totalByEmotion` | Map<String,int>                                                                 | вкладка «Эмоции» → точность по каждой эмоции              |
| `correctByModule` / `totalByModule`   | Map<String,int>                                                                 | главный экран (карточка модуля «5/7 · 71%»)                        |
| `reactionTimes`                         | List `<int>`, последние 50                                           | вкладка «Обучение» → среднее RT                                        |
| `avgReactionByModule`                   | Map<String,double>, derived из `reactionByModule`                           | вкладка «Обучение» → RT по модулям                                   |
| `reactionByModule`                      | Map<String,List `<int>`>, последние 30 на модуль             | источник для `avgReactionByModule`                                                 |
| `sessionLog`                            | Map<date,int> ответов в день, обрезается до 30 дней | вкладка «Обучение» → график активности 14 дней            |
| `difficulty`                            | Map<String,int> 1–5 на модуль                                          | вкладка «Обучение» → уровень сложности                        |
| `completedModules`                      | Set `<String>`                                                                | главный экран (иконка ✓ и переход на restart-экран)           |
| `recentAnswers`                         | Map<String,List `<bool>`> окно 5                                          | внутренняя адаптация сложности, в UI не отображается |

Все поля выше **персистятся** в SharedPreferences. Это значит, что после перезапуска приложения исторические метрики (активность по дням, среднее время реакции, окно адаптации сложности) сохраняются — а не сбрасываются, как было до задачи #1 в ROADMAP. `avgReactionByModule` единственное derived — пересчитывается из `reactionByModule` при `load()`, отдельным ключом не хранится.

#### Из `DiagnosticModel` (хранится на профиль)

| Поле                                | Тип                 | Где видно                                                                                              |
| --------------------------------------- | ---------------------- | -------------------------------------------------------------------------------------------------------------- |
| `preSession` / `postSession`        | `DiagnosticSession?` | вкладка «Диагностика» → сводная карточка + сравнение              |
| `accuracy` (на сессию)        | double 0–1            | сводная карточка                                                                                |
| `avgReactionMs` (на сессию)   | double                 | сводная карточка                                                                                |
| `accuracyForEmotion(emotion)`         | double 0–1            | таблица «Динамика по эмоциям» (только при наличии обеих фаз) |
| `answers[]` каждой сессии | `DiagnosticAnswer[]` | основа всех CSV-экспортов                                                                   |

### Структура вкладок

| Вкладка                   | Карточки                                                                                                                                                                                                                                                                    | Опирается на              |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| **Обучение**       | Общая статистика (дней/ответов/% верных), активность 14 дней, прогресс по модулям, уровень сложности, среднее время реакции, инфо-блок про метод лестницы | `ProgressModel`                    |
| **Диагностика** | Сводная карточка pre/post (точность + RT), таблица «динамика по эмоциям» (если есть обе фазы), bar chart точности, инфо-блок про T-критерий Вилкоксона                         | `DiagnosticModel`                  |
| **Эмоции**           | Точность по каждой из 6 эмоций (отсортировано от худшей к лучшей), карточка рекомендаций со списком слабых эмоций (минимум 3 ответа)                                      | `ProgressModel.accuracyForEmotion` |

> Точность на вкладке «Эмоции» — **кумулятивная по обучению**, не по диагностике. Если ребёнок прошёл pre-тест, но ещё не занимался — вкладка покажет «Данных пока нет». Точность диагностики по эмоциям видна на вкладке «Диагностика» в таблице «Динамика».

## Статистика проекта

Полный разбор всей математики исследования: дизайн анализа, что означает каждая переменная, какой критерий для какого сравнения, как метрики считаются в коде, и где лежат данные. Это справочник «на одном экране» — чтобы не выводить формулы заново.

### Дизайн анализа

Квазиэксперимент с **двумя группами** и измерением **до/после**:

- **Экспериментальная** — проходит обучающие модули между pre- и post-диагностикой.
- **Контрольная** — только диагностика pre/post, без обучения. Нужна, чтобы отделить эффект приложения от естественного развития и эффекта повторного тестирования.

Измеряются **две методики** (`DiagnosticMeasure`), раздельно:

- **«Эмоциональные лица» (`faces`)** — перцептивный компонент: эмоция по фото. 18 заданий (3/эмоция), балл 0–18.
- **«Социальные истории» (`stories`)** — контекстный компонент: эмоция по ситуации. 6 заданий (1/эмоция), балл 0–6.

Единицы: точность — доля 0–1 (или %), баллы — целые, время реакции (RT) — миллисекунды.

### Глоссарий переменных

| Обозн.   | Что это                                           | На пальцах                                                                                                                          |
| ------------- | ------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `N` / `n` | объём выборки                               | сколько участников (n=20 эксп, n=7 контр)                                                                          |
| `M`         | среднее арифметическое             | сумма ÷ число участников                                                                                                |
| `SD`        | стандартное отклонение             | разброс вокруг среднего (мал = все похожи)                                                                  |
| `Mdn`       | медиана                                          | «средний по списку» участник; устойчивее к выбросам                                              |
| `RT`        | reaction time                                           | мс на ответ; меньше = автоматичнее навык                                                                     |
| `п.п.`    | процентные пункты                       | разница двух процентов (32 %→73 % =**+41 п.п.**, не «+41 %»)                                                |
| `p`         | уровень значимости                     | вероятность, что результат — случайность.`<0,05` значимо, `<0,001` высоко значимо |
| `T`         | статистика Вилкоксона               | сумма рангов;`T=0` = никто не стал хуже                                                                          |
| `Z`         | стандартизованная статистика | по ней считается `p`; большое \|Z\| → крошечное p                                                           |
| `U`         | статистика Манна-Уитни              | аналог T для сравнения двух групп                                                                                 |
| `χ²`      | хи-квадрат (Мак-Немар)                 | статистика для сдвигов в бинарных данных                                                                  |
| `b / c`     | дискордантные пары                     | `b` = верно→неверно, `c` = неверно→верно; совпавшие критерий игнорирует             |
| `r`         | размер эффекта                             | `r = Z/√N`; шкала Коэна: 0,1 мал / 0,3 средн / 0,5 больш                                                           |

### Какой критерий для какого вопроса

| Вопрос                                                                                    | Критерий                                                                                     | На каких колонках CSV                                                                                    |
| ----------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| Стала ли**одна группа** лучше? (within)                             | **Вилкоксон** знаковых рангов (T, Z, p, r)                              | `pre_accuracy` vs `post_accuracy` (и `*_stories_accuracy`, RT)                                                   |
| **Две группы** различаются по приросту? (between)           | **Манн-Уитни** (U, p)                                                                 | прирост `post−pre` эксп vs контр                                                                     |
| Сдвиг по**одной эмоции** значим? (бинарный)               | **Мак-Немар** (χ² Йейтса, пары b/c, точный биномиальный) | `study_detailed`: пары pre↔post по позиции задания внутри эмоции (faces, ~60 пар) |
| Насколько эффект**большой**, а не просто значимый? | размер эффекта `r`                                                                    | из Z Вилкоксона                                                                                             |

Все критерии **непараметрические** — выбор обоснован малой клинической выборкой и порядковыми/бинарными шкалами без допущения нормальности.

Коротко: **Вилкоксон** = одна группа во времени, **Манн-Уитни** = две группы между собой, **Мак-Немар** = одна группа во времени по одному бинарному признаку.

### Как метрики считаются в коде

Источник — `DiagnosticSession` (`lib/models/diagnostic_model.dart`). Все перцептивные метрики — **только по faces**:

- `accuracy` = `facesCorrect / facesTotal`;  `storiesAccuracy` = `storiesCorrect / storiesTotal`.
- `accuracyForEmotion(e)` = верных faces по эмоции `e` ÷ 3.
- `avgReactionMs` / `avgReactionForEmotion(e)` — среднее RT (faces).
- `facesCorrect/Total`, `storiesCorrect/Total` — сырые счётчики (= баллы 0–18 и 0–6).

Сводный CSV — это ровно эти геттеры, разложенные по колонкам (`_summaryRowTail`).

## Соответствие ВКР — точные данные для главы 4

### Персонаж-помощник

Персонаж есть. **Кот Апельсин** (эмодзи 🐱, имя задано в `onboarding_screen.dart:25`). Реализован как:

- Онбординг-слайд (знакомство с механикой)
- Виджет `HelperCat` — речевой пузырь с подсказкой к заданию (показывается до ответа, скрывается после)
- Строка поощрения на экране награды

Нарратива и сюжетной линии нет. Роль — функциональный помощник-подсказчик.

### Система наград (`reward_screen.dart`)

Статичный экран без игровой механики. Показывает:

- Текстовое поздравление с именем завершённого модуля и упоминанием персонажа Апельсина
- Анимированный трофей 🏆: spring-появление (`elasticOut`, 600 мс)
- Три последовательные звезды ⭐⭐⭐: появление с `easeOutBack` + золотой shimmer
- Кнопку возврата на главный экран

### Модуль 1 — точная структура заданий

- Пул: 18 заданий (6 эмоций × 3 фото); активный подпул фильтруется по `difficultyLevel <= ((maxDiff + 1) ~/ 2 + 1)`
- Состав экрана задания: подсказка `HelperCat` → вопрос → сетка карточек (2 колонки)
- Карточки: 1 целевая эмоция + (N−1) случайных дистракторов; N ∈ {2, 4, 6}
- Фото через `ColorFilter` (матрица смягчения насыщенности)
- После ответа: `HelperCat` скрывается (`AnimatedSize`), появляются `FeedbackBanner` + `EmotionInfoCard` + кнопка «Дальше»

### Адаптивная сложность — точные пороги

- **Верхний порог:** accuracy > 85% → уровень +1
- **Нижний порог:** accuracy < 70% → уровень −1
- **Окно:** скользящее, максимум 5 последних ответов
- **Минимум для срабатывания:** 3 ответа в окне (при меньшем количестве пересчёт не производится)
- **Диапазон уровней:** 1–5, независимо для каждого из трёх модулей
- Источник: `progress_model.dart`, методы `_adaptDifficulty` (строки 159–170) и `recordAnswer` (строки 126–156)
